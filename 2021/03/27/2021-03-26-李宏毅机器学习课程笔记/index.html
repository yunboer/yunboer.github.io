李宏毅机器学习课程笔记

引言

  1. “机器学习就是自动找函数。”：

​ 例如 语音辨识，输入语音，输出文字。

​ 图像识别，输入图片，输出tag

​ 阿法狗，输入棋局，输出坐标

  1. 找什么样的函数？例如 regression（输出数值），binary classification（二分），multi-class classification（多分类），generation（输出复杂的信息）
  2. 如何告诉机器要找的函数？例如 supervised learning（有label的数据+函数的loss用于判断函数好不好），reinforcement learning（阿法狗下了很多步骤之后赢了获得reward，自行判断哪一步应该怎么下），unsupervised learning（给很多data，没有目标，比如分类问题）
  3. 机器如何找出要找的函数？（a.）network architecture 例如 给出函数范围（linear线性，neural network神经网络（RNN，CNN））(b.)找出最优函数例如 gradient descent梯度下降
  4. 前沿研究？Explainable AI（说明输出结果的原因）、Adversarial Attack（恶意攻击系统，加入针对AI的干扰）、Net Compression（模型太大，要将模型缩小）、Anomaly Detection（AI知道：我不知道）、Domain Adversarial Learning（训练集和测试集不同如何处理）、Meta Learning（学习如何学习，学习写程序算法的能力）、Life-long Learning（终身学习）

Regression回归

  1. 例如 股票预测（股票未来信息=f(股票历史信息)），无人车（方向盘角度=f（周围环境信息）），推荐系统（可能性=f（使用者A，商品B））
  2. 预测宝可梦cp值，进化后cp值=f（宝可梦类型，hp，体重，身高）
  3. 第一步：model： y=b+w \ Xcp. (linear，x是feature）*
  4. 第二步：上标表示一个元素，下标表示元素中的内容，头顶尖尖表示预测值；定义一个loss function（输入是一个待测模型，输出模型的不好程度），L（f）=L（b，w），简单定义为误差的平方和。
  5. 第三步：找到让L最低的函数，Gradient Descent（梯度下降法），找微分然后根据微分的大小调节参数，变化量=-η*dL/dw（w是参数），只能找到local optimal（局部最优，当然线性回归没有局部最优）。两个参数就是分别求两个偏微分即可。梯度是一个向量（就是偏导组成的向量）。
  6. 第四步：training data和testing data，复杂的式子可能会overfitting。
  7. 第五步：Regularization（正则化），L=误差平方+λ（参数的平方）（这里不用考虑bais（常数项））让参数尽可能小，模型更加平滑，不易于overfitting ，然后调节λ增加，training的误差增大，直到最小的testing误差

Basic Concept

  1. 对于f的估计就是一个随机变量的估计，误差有两个来源
  2. 调节error来源：来自”bias（是否有偏）”或者来自”variance（方差）”
  3. 随机变量的平均值和期望是unbiased（无偏估计）
  4. 在regression回归中，f模型越简单，variance误差越小。
  5. 如果真实的model比较复杂，则简单的模型一定有bias，取复杂度越高bias越小。
  6. 也就是单次试验来说，复杂的model可能会偏差较大，但是多次试验的平均model是接近真实的。这是由于简单的model函数可取区域比较小，可能就不包含target。。
  7. 复杂的函数误差大：overfitting（原因：variance；表现：testing误差大；方案：增大数据量）；简单的函数误差大underfitting（原因bias；表现training误差大；方案：修改model）
  8. regularization：变平滑，变小variance误差，伤害bias。。
  9. 只考虑当前的testing set，通过当前的testing set评判model好坏，可能会不是真实的最优。
  10. 方案1：training set分成两个组：一个training；一个validation选择好的model（只是选择model，不确定参数）；再用全部的training set训练；最后用testing得到error才是真实error。
  11. 其实一般人都会根据testing调节函数，这样结果一般不好，但是方便发paper。
  12. 方案2：training set分成三个组：分别取两个training一个validation，求每个model的平均error，再选定model

Gradient Descent（梯度下降法）

  1. L是loss function
  2. 首先选择一组起始参数θ，计算偏微分，原参数减去偏微分乘η（一个参数learning rate）
  3. 反复减去这个偏差（梯度下降）
  4. learning rate需要谨慎调节，右图是随着梯度下降和loss值大小的趋势图，最好的情况是调节到红色的线
  5. learning rate一开始大，后来变小：adagrad，其中w是参数，η是learning rate（每次都变），σ是一个调节learning rate的参数（反映了二次微分，而二次微分可以解决不同参数距离对比问题），g是偏微分
  6. stochastic gradient descent：选一个example的loss，依次把所有example都下降完（右图，一次走20步，左图一次走一步）。
  7. feature scaling（特征的范围化成一致）
  8. 限制：local minima，微分是0不一定是极值。

Classification（分类）

  1. 用回归代替分类（二分类为例）：分类1是1，分类2是-1
  2. 如果直接将分类问题线性化，出现问题
  3. 解决方案1：修改线性回归的loss函数，直接改成预测分类错误的次数。不可以用梯度下降法（用perceptron或者svm求解最佳模型）
  4. 解决方案2：Generative Model（生成模型）使用贝叶斯公式
  5. 需要计算4个概率：计算两个PC在数据集中直接用频率做。
  6. 假设水系神奇宝贝的防御和特防满足一个gaussian distribution（高斯分布也就是正态分布），根据原来的training data用Maximum Likelihood（求出一个得到全部 training data概率最高的分布函数（极大似然估计））求出分布函数，就可以算出抽一个水系正好是新宝可梦的概率
  7. 问题就解决了
  8. 按照概率0.5划线
  9. Mofifying Model：强制两个分布的方差相同，分界线变成直线
  10. 分类三步走：model是一个概率函数，0.5为分界线；training就是找概率分布的参数（极大似然）；找到最好的函数
  11. 其实分布可以自己选择决定的：例如Naive Bayes Classifier朴素贝叶斯（假设高斯分布都是独立的，也就是各个feature相互独立）
  12. 这就是sigmoid函数，全体起立！

Logistic Regression（Logistic回归）

  1. 前一节推导sigmoid函数就是几率的函数（值域是0到1）。
  2. z是一个线性的函数z=wx+b
  3. training是根据sigmoid函数产生，求某个w和b产生这些training data的概率，极大似然估计。
  4. 求L(w,b)=f(x1)f(x2)(1-f(x3))…最大值也就是求-lnL(w,b)最小值，变成加法，这里将分类数据化（图中有一处错：110，不是101）。
  5. 这就是Loss Function，这是training的过程。
  6. 接下来，gradient descent和linear公式一样
  7. 不可以使用square error
  8. Discriminative判别模型（Logistic） v.s. Generative生成模型（Gaussian）其实他们的model一模一样，都是w和b的参数。找到的最优解不同，Discriminative在data多时更好，Generative在data少的时候能赢。
  9. 多个class情况：每个class一个z
  10. Limitation of Logistic Regression：logistic的缺陷，可以用Feature Transformation空间转换。这是人工的算法，如果想要机器自己产生，则可以用Logistic叠加，就是Neural Network

Ensemble（集成学习）

bagging

  1. 不同模型的输出平均，得到合适的模型。

  2. 有N个training data，每次取N’（一般就是N，有放回抽取）个data组成一个新的data set，对于这几个data set做learning获得多个模型，最后在testing data上做平均或者投票。

  3. model复杂，担心overfitting做bagging（例如decision tree）

  4. decision tree：根据输入建立一棵树，最后得到分类结果（几个分支，什么时候停止分支，什么标准来做分支，分支条件有哪些），这种方法很容易overfit，如果层数很深，可以让training data到100%正确。

  5. Random Forest：Decision Tree容易overfitting，传统的bagging每棵树都差不太多，改善效果不明显。在每次产生一棵Decision Tree的时候都随机决定哪些faeture或者问题是不可以用的。在投票就是Random Forest。

  6. Bagging的validation可以用Out-of-bag，可以不把data切分成training set和validation set，可以把没有用到某个data的modelbagging一下，然后用这个data去test他。

boosting

  1. 如果有一个模型可以达到50%以上正确率的分类器，boosting可以达到100%
  2. 首先找一个classifier
